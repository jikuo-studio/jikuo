## 1. Product Meaning

This slice closes the smallest evidence loop:

- CORE-10 made required evidence visible.
- CORE-11 made compact policy evidence persistence proposable.
- CORE-12 lets a later check read that persisted evidence back from a task session.

Visible business value:

- the user does not need the desktop agent to remember prior chat context
- persisted process evidence becomes reusable by code-level evaluation
- task-session evidence remains compact and explicit
- the proposal still performs no durable write

Important boundary:

- this is evidence ingestion, not action execution
- this is report-only, not a gate
- evidence source must be an explicit task session id
- no broad filesystem scan or hidden evidence discovery is introduced

## 3. Kernel Compatibility

This slice preserves kernel compatibility by:

- using `evidence_kind: policy_evidence:<type>` as the task-session evidence ingestion prefix
- parsing compact `source_ref` fields for `policy_ref`, `action_ref`, and `requirement_ref`
- keeping task-session evidence snapshots as evidence source, not policy source
- preserving policy store as the active policy source
- reporting `evidence_source_refs` instead of copying raw session bodies
- keeping missing evidence reports unchanged when persisted evidence is absent or non-matching
- keeping gate behavior deferred

## 9. Delivery Criteria

- `policy_store.py evaluate --event task_start --task-session-id task_policy_evidence_probe ...` reports satisfied required evidence from persisted task-session evidence.
- `agent_flow.py propose --event policy_evidence_check --policy-event task_start --session-id task_policy_evidence_probe ...` returns `jikuo.agent_flow_proposal.v1`.
- Proposal includes `CAP-POLICY-EVIDENCE-INGEST-01`.
- Proposal includes task-session id and evidence source refs.
- Proposal does not modify the target task-session file.
- Missing or unsafe session ids produce warnings/refusals without writes.
- Tests and checker smoke pass.
