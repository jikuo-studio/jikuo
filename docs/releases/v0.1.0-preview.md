# JIKUO v0.1.0-preview

Release date: 2026-06-13

JIKUO v0.1.0-preview is the first source-available noncommercial preview
release. It is intended for first users who want to inspect the product loop,
run local first-use diagnostics, activate starter governance policies, and see
runtime evidence without relying on private maintainer-local project knowledge.

This is not a production or commercial-use release. The repository is licensed
under the PolyForm Noncommercial License 1.0.0; commercial use, hosted service
use, production business use, and brand/IP terms require a separate written
agreement.

## What Is Included

- CLI entry points for `jikuo`, `jikuo doctor`, `jikuo configure`,
  `jikuo policy-management`, `jikuo studio`, and `jikuo show`.
- A local stdio MCP server exposing the current 24-tool MVP surface for status,
  cards, no-write proposals, activation settings, policy management, starter
  policies, Sampling semantic-provider probing, and guarded writes.
- A read-only Studio console backed by the Studio global-status read model.
- First-run configuration review for activation settings, project context,
  starter policies, instruction files, runtime visibility, and diagnostics.
- Document Rules for explaining and previewing project-local document mounts.
- Starter policy templates and guarded preview/apply paths for baseline policy
  activation.
- Policy Trace, Document Trace, semantic intent evidence, turn anchors, missing
  evidence classification, current limitations, and Work receipt checkpoint
  visibility.
- A non-private demo starter project under `examples/demo_project/`.

## First-Run Path

```powershell
cd <workspace>
git clone https://github.com/jikuo-studio/jikuo.git
cd jikuo
py -3 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e ".[dev]"
.\.venv\Scripts\jikuo.exe doctor --format markdown
.\.venv\Scripts\jikuo.exe studio serve --host 127.0.0.1 --port 8765
```

The full first-run guide is
[`docs/user/getting-started.md`](../user/getting-started.md). Users should
complete activation settings review, starter policy activation, Document Rules
review, and a small governed work turn before treating a project as ready.

## Known Limits

- JIKUO does not perform semantic judgment by itself. Host AI supplies compact
  semantic intent when available; JIKUO records, merges, triggers policies, and
  displays evidence.
- Many policies are report-only. A visible missing evidence report is not
  automatically a blocking gate unless a policy explicitly participates in a
  gate.
- Runtime history is not yet a full DATA-01 event ledger.
- Observed-read evidence remains limited; write-side receipt evidence is more
  mature than read-side proof.
- Document Trace can report no comparable trace when structured round-level
  evidence is unavailable.
- Some final-response and completion applicability evidence remains
  policy-specific backlog.
- Strict mounted behavior depends on a host adapter. Instruction-only or
  MCP-only setup must not be presented as guaranteed pre-turn execution.
- Policy conditions currently compose as AND; generic OR condition groups such
  as `any_of` or `one_of` are not implemented yet.

See [`docs/user/limitations.md`](../user/limitations.md) for the complete
user-facing limitations guide.
