# Release Checklist

Use this checklist before saying the demo workflow is complete.

- The project context is present at `.jikuo/project_context.yaml`.
- The brief, decision log, and checklist are visible in Document Rules.
- Starter policy activation is previewed before any guarded apply.
- Any generated release note names the evidence it used.
- Missing evidence is explained as a current product boundary, user setup gap,
  or real workflow gap.
